# 🔺 Метод Нелдера — Мида

> Метод деформируемого многогранника — метод безусловной оптимизации функции от нескольких переменных, **не использующий производных**. Суть метода заключается в последовательном перемещении и деформировании симплекса вокруг точки экстремума.

> ⚠️ Метод находит **локальный** экстремум и может «застрять» в одном из них. Если требуется найти глобальный экстремум — следует пробовать разные начальные симплексы.

---

## 📐 Алгоритм

Реализованный программно алгоритм выполняет следующие шаги:

### 1. Инициализация симплекса

Задаётся начальная точка **x₀** и строится симплекс из **n+1** вершин. Каждая следующая вершина получается смещением i-й координаты на δᵢ:

$$\delta_i = 0.1 \cdot \max(1,\ |x_0^{(i)}|)$$

```python
Simplex.from_point(x0, step=0.1)
```

---

### 2. Сортировка вершин

Вершины упорядочиваются по значению функции:

$$f(\mathbf{x}_l) \leq f(\mathbf{x}_g) \leq f(\mathbf{x}_h)$$

| Обозначение | Название | В коде |
|---|---|---|
| **x_l** | лучшая (least) | `simplex.best` |
| **x_g** | средняя (good) | `simplex.second_worst` |
| **x_h** | худшая (high) | `simplex.worst` |

```python
simplex._sort()
```

---

### 3. Вычисление центроида

Находится центр масс всех вершин, **кроме худшей**:

$$\bar{\mathbf{x}} = \frac{1}{n} \sum_{i=1}^{n} \mathbf{x}_i, \quad \mathbf{x}_i \neq \mathbf{x}_h$$

```python
simplex.centroid_excluding_worst()
```

---

### 4. Отражение

Худшая вершина отражается через центроид:

$$\mathbf{x}_r = \bar{\mathbf{x}} + \alpha(\bar{\mathbf{x}} - \mathbf{x}_h), \quad \alpha = 1$$

```python
xr = xc + self._cfg.alpha * (xc - xh)
```

---

### 5. Анализ результата — выбор операции

```
f_r < f_l  →  Растяжение
f_l ≤ f_r < f_g  →  Принятие отражения
f_g ≤ f_r < f_h  →  Внешнее сжатие
f_r ≥ f_h  →  Внутреннее сжатие
сжатие не помогло  →  Редукция
```

#### 🟢 Растяжение
Если **f_r < f_l** — направление выбрано удачно, пробуем увеличить шаг:

$$\mathbf{x}_e = \bar{\mathbf{x}} + \gamma(\mathbf{x}_r - \bar{\mathbf{x}}), \quad \gamma = 2$$

Принимается лучшая из **x_r** и **x_e**.

```python
xe = xc + self._cfg.gamma * (xr - xc)
```

#### 🔵 Принятие отражения
Если **f_l ≤ f_r < f_g** — новая точка лучше двух прежних:

$$\mathbf{x}_h \leftarrow \mathbf{x}_r$$

```python
simplex.replace_worst(Vertex(xr, fr))
```

#### 🟡 Внешнее сжатие
Если **f_g ≤ f_r < f_h**:

$$\mathbf{x}_c = \bar{\mathbf{x}} + \beta(\mathbf{x}_r - \bar{\mathbf{x}}), \quad \beta = 0.5$$

```python
xc_out = xc + self._cfg.beta * (xr - xc)
```

#### 🟠 Внутреннее сжатие
Если **f_r ≥ f_h**:

$$\mathbf{x}_c = \bar{\mathbf{x}} + \beta(\mathbf{x}_h - \bar{\mathbf{x}}), \quad \beta = 0.5$$

```python
xc_in = xc + self._cfg.beta * (xh - xc)
```

#### 🔴 Редукция
Если сжатие не улучшило результат — все вершины, кроме лучшей, притягиваются к **x_l**:

$$\mathbf{x}_i = \mathbf{x}_l + \sigma(\mathbf{x}_i - \mathbf{x}_l), \quad \sigma = 0.5, \quad i = 2 \ldots n+1$$

```python
simplex.shrink_towards_best(objective)
```

---

### 6. Проверка сходимости

Алгоритм останавливается при выполнении **хотя бы одного** условия:

$$\text{std}\bigl(f(\mathbf{x}_1),\ldots,f(\mathbf{x}_{n+1})\bigr) < \varepsilon \quad \text{или} \quad \text{diam}(\text{simplex}) < 100\varepsilon$$

где **ε = 10⁻⁸**.

```python
_has_converged(simplex)
```

## 🧪 Примеры

### Пример 1 — Функция Розенброка

$$f(x_0, x_1) = (1 - x_0)^2 + 100(x_1 - x_0^2)^2$$

**Аналитическое решение** (через частные производные):

$$\frac{\partial f}{\partial x_0} = 2(x_0 - 1) - 400x_0(x_1 - x_0^2) = 0$$

$$\frac{\partial f}{\partial x_1} = 200(x_1 - x_0^2) = 0 \implies x_1 = x_0^2$$

Подставляя во второе уравнение: $2(x_0 - 1) = 0 \implies x_0 = 1,\ x_1 = 1$

$$\boxed{x^* = (1,\ 1), \quad f(x^*) = 0}$$

**Проверка программой** (x0 = `-1.5, 2.0`):

![Розенброк](https://github.com/user-attachments/assets/6a6deca7-1333-43bf-bb6e-59b81be20cb1)

Все решения сходятся к точке **(1, 1)**, значение функции **≈ 0**. Алгоритм корректно реализован.

> **Почему 94 итерации?** Функция Розенброка имеет узкую изогнутую долину. Матрица Гессе в точке минимума имеет число обусловленности **≈ 200**, что вынуждает симплекс делать много мелких шагов вдоль параболической траектории.

---

### Пример 2 — Функция сферы

$$f(x_0, x_1) = x_0^2 + x_1^2$$

$$\frac{\partial f}{\partial x_0} = 2x_0 = 0, \quad \frac{\partial f}{\partial x_1} = 2x_1 = 0 \implies x^* = (0,\ 0)$$

$$\boxed{x^* = (0,\ 0), \quad f(x^*) = 0}$$

**Проверка программой** (x0 = `0, 0`):

![Сфера](https://github.com/user-attachments/assets/eaa7d73a-6e70-49ba-8241-4c584dbd8675)

Алгоритм продемонстрировал ожидаемое поведение — сошёлся в нуле.

> **Почему 20 итераций для простой функции?** Минимум найден уже на первой итерации, однако алгоритм продолжает работу до тех пор, пока симплекс не схлопнется до размера меньше tolerance. При начальном диаметре **0.1** и требуемом **10⁻⁶** необходимо **log₂(10⁵) ≈ 17** операций сжатия.
