# The-Nelder-Meade-method
Метод Нелдера - Мида. Суть метода заключается в последовательном перемещении и деформировании симплекса вокруг точки экстремума.
Метод находит локальный экстремум и может «застрять» в одном из них. Если всё же требуется найти глобальный экстремум, можно пробовать выбирать другой начальный симплекс.
Реализованный программно алгоритм выполняет следующие шаги:
1. Инициализация симплекса.
Задаётся начальная точка x₀ и строится симплекс из n+1 вершин. Каждая следующая вершина получается смещением i-й координаты на δᵢ:
δᵢ = 0.1 · max(1, |x₀ᵢ|)
В коде: Simplex.from_point(x0, step=0.1)

2. Сортировка вершин.
Вершины упорядочиваются так, что: f(x_l) ≤ f(x_g) ≤ f(x_h), где x_l — лучшая (least), x_g — средняя (good), x_h — худшая (high).
В коде: simplex._sort(), обозначения — simplex.best, simplex.second_worst, simplex.worst

3. Вычисление центроида
Находится центр масс всех вершин кроме худшей: x̄ = (1/n) · Σᵢ₌₁ⁿ xᵢ,   xᵢ ≠ x_h
В коде: simplex.centroid_excluding_worst()

4. Отражение.
Худшая вершина отражается через центроид: x_r = x̄ + α(x̄ − x_h),   α = 1
В коде: xr = xc + self._cfg.alpha * (xc - xh)

5. Анализ результата — выбор операции.
Растяжение — если f_r < f_l, направление выбрано удачно, пробуем увеличить шаг:
x_e = x̄ + γ(x_r − x̄),   γ = 2
Принимается лучшая из x_r и x_e.
В коде: xe = xc + self._cfg.gamma * (xr - xc)

Принятие отражения — если f_l ≤ f_r < f_g, новая точка лучше двух прежних:
x_h ← x_r
В коде: simplex.replace_worst(Vertex(xr, fr))

Внешнее сжатие — если f_g ≤ f_r < f_h:
x_c = x̄ + β(x_r − x̄),   β = 0.5
В коде: xc_out = xc + self._cfg.beta * (xr - xc)

Внутреннее сжатие — если f_r ≥ f_h:
x_c = x̄ + β(x_h − x̄),   β = 0.5
В коде: xc_in = xc + self._cfg.beta * (xh - xc)

Редукция — если сжатие не помогло, все вершины кроме лучшей притягиваются к x_l:
xᵢ = x_l + σ(xᵢ − x_l),   σ = 0.5,   i = 2..n+1
В коде: simplex.shrink_towards_best(objective)

6. Проверка сходимости
Алгоритм останавливается при выполнении хотя бы одного условия:
std(f(x₁),...,f(x_{n+1})) < ε   или   diameter(simplex) < 100ε
где ε = 10⁻⁸. В коде: _has_converged(simplex)

Проверка работы алгоритма. Возьмём данную функцию:
f(x0​,x1​)=(1−x0​)^2+100(x1​−x0^2​)^2. Решим её вручную, чтобы сравнить получившийся ответ с результатами, получнными программой 
Ищем частные производные
х0:2(x0​−1)−400x0​(x1​−x0^2​)
x1:200(x1​−x02​)
Приравниваем к нулю => получается следующая система:
1. 2(x0​−1)−400x0​(x1​−x0^2​)=0
2. 200(x1​−x02​)=0 =>  x1=x0 подставляем в 1. 2(x0​−1)=0 => x0​=1 x1=1
Подставив эти значения в исходную функцию получим что f(x0, x1) = 0
